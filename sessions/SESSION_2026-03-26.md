# Сессия 2026-03-26 — VoiceClaude

## Что обсуждали
Задача: сделать VoiceClaude рабочим на любом виртуальном рабочем столе Windows 10/11
и поддерживающим запуск нескольких независимых экземпляров.

## Принятые решения
- Добавить `--instance N` и `--config FILE` аргументы командной строки
- Каждый экземпляр получает уникальное имя: `VoiceClaude` (0) или `VoiceClaude #N` (1+)
- Каждый экземпляр может читать свой config.json (разные хоткеи → нет конфликтов)
- Закрепление окна на всех виртуальных рабочих столах через Windows COM API:
  `IVirtualDesktopPinnedApps::PinWindow` через `ImmersiveShell` + `IServiceProvider`
- Автозагрузка в реестр пишет прямой путь к pythonw.exe + --instance arg (не VBS)

## Что сделали
- `main.py`: добавлены argparse, INSTANCE_ID, instance-aware APP_NAME
- `main.py`: CONFIG_PATH теперь зависит от --config аргумента
- `main.py`: функция `pin_to_all_desktops(hwnd)` — COM API через ctypes, без доп. зависимостей
- `main.py`: функция `_launch_cmd()` — строит команду автозагрузки с instance-аргументами
- `main.py`: `set_autostart()` обновлён для мульти-экземпляров
- `main.py`: `VoiceWindow._pin_to_all_desktops()` вызывается через after(600) при старте
- `VoiceClaude.vbs`: принимает дополнительные аргументы (WScript.Arguments) и передаёт их main.py

## Как запускать несколько экземпляров

| Файл | Действие |
|------|----------|
| `VoiceClaude.vbs` | Экземпляр 0 (основной, Caps Lock) |
| `VoiceClaude_1.vbs` | Экземпляр 1 (двойной клик) |
| `VoiceClaude_N.vbs` | Создать: `create_instance.bat N` |

**Порядок создания нового экземпляра:**
1. Запустить `create_instance.bat` → ввести номер (например 2)
2. Создаётся `VoiceClaude_2.vbs` + `config_2.json`
3. Открыть `config_2.json`, изменить `hotkey` на другую клавишу (например `f9`)
4. Двойной клик на `VoiceClaude_2.vbs`

`load_config()` в main.py автоматически создаёт `config_N.json` из `config.json` если файл не найден.
